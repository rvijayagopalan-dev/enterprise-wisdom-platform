# Domain 05: Enterprise Architecture

## Purpose
Define current and future-state enterprise architecture, govern technology decisions, and produce actionable transformation roadmaps.

## Sub-Domains

| Sub-Domain | Description |
|------------|-------------|
| [Current State Architecture](./current-state-architecture/README.md) | Document and assess the as-is architecture |
| [Future State Architecture](./future-state-architecture/README.md) | Design the to-be architecture aligned to strategy |
| [Architecture Governance](./architecture-governance/README.md) | Standards, review boards, and compliance controls |
| [Technology Rationalization](./technology-rationalization/README.md) | Portfolio rationalization and technology decommissioning |
| [Architecture Decision Records](./architecture-decision-records/README.md) | Capture and preserve architecture decisions with rationale |

## Frameworks
- **TOGAF** — The Open Group Architecture Framework
- **ArchiMate** — Architecture modeling notation
- **Zachman** — Enterprise architecture classification framework

## Outputs
- Reference Architectures
- Transformation Roadmaps
- Architecture Decision Records (ADRs)
- Technology Standards Catalog

## AI Advisor
**Chief Architect Advisor** — generates architecture recommendations, evaluates technology options, and produces transformation blueprints.
