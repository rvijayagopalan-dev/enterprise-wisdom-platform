# Sub-Domain: Data Products

## Purpose
Design, build, and manage data as discoverable, trusted, and reusable products consumed across the enterprise.

## Data Product Attributes
- Discoverable — listed in data catalog
- Addressable — accessible via standard interfaces
- Trustworthy — quality SLAs and lineage
- Self-describing — documented schema and semantics
- Interoperable — standard formats and contracts

## Outputs
- Data Product Catalog
- Data Product Specifications
- Data Product Quality Metrics
