# Domain 06: Data & AI Intelligence

## Purpose
Define enterprise data strategy, design AI architectures, and govern data and AI initiatives across the organization.

## Sub-Domains

| Sub-Domain | Description |
|------------|-------------|
| [Data Strategy](./data-strategy/README.md) | Define enterprise data vision, principles, and roadmap |
| [Data Mesh](./data-mesh/README.md) | Federated data ownership and product-thinking for data |
| [Data Products](./data-products/README.md) | Build and manage data as reusable products |
| [Data Governance](./data-governance/README.md) | Data quality, lineage, ownership, and compliance |
| [AI Strategy](./ai-strategy/README.md) | Define AI vision, use case prioritization, and adoption roadmap |
| [Agentic AI](./agentic-ai/README.md) | Design autonomous AI agents and multi-step AI workflows |
| [Multi-Agent Systems](./multi-agent-systems/README.md) | Architect systems of cooperating AI agents |
| [Knowledge Graphs](./knowledge-graphs/README.md) | Enterprise knowledge representation and reasoning |
| [AI Governance](./ai-governance/README.md) | Risk, ethics, compliance, and oversight for AI systems |

## Outputs
- Enterprise AI Blueprint
- Data Operating Models
- Agent Architectures
- Enterprise Knowledge Graph Design

## AI Advisor
**Chief Data Officer Advisor** — guides data strategy, mesh design, and governance.
**AI Governance Advisor** — oversees AI risk, ethics, and regulatory compliance.
