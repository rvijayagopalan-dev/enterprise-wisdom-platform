# Domain 04: Operating Model

## Purpose
Assess current operating models and design target operating models across people, process, technology, governance, and metrics.

## Sub-Domains

| Sub-Domain | Description |
|------------|-------------|
| [Current State Assessment](./current-state-assessment/README.md) | Evaluate existing operating model dimensions |
| [Target Operating Model](./target-operating-model/README.md) | Design the future-state TOM blueprint |
| [Product Operating Models](./product-operating-models/README.md) | Operating models for product-led organizations |
| [Platform Operating Models](./platform-operating-models/README.md) | Operating models for internal platform teams |
| [AI Operating Models](./ai-operating-models/README.md) | Operating models governing AI delivery and governance |

## Dimensions
- **People** — roles, structure, skills, culture
- **Process** — workflows, decision rights, handoffs
- **Technology** — tools, platforms, automation
- **Governance** — policies, controls, accountability
- **Metrics** — KPIs, SLAs, performance targets

## Outputs
- TOM Blueprints
- Workforce Transformation Plans
- Organizational Design Models
